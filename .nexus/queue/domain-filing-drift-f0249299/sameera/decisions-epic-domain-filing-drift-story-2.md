## 2026-07-19 — Story 2 ships as a new deterministic tool, not a prompt edit
- **Choice:** Implement the advisory as `libs/portable-tools/src/drift-advisory.ts` under the parity+fingerprint gate; the prompt only invokes it.
- **Why:** The decision record makes the advisory deterministic, integer-exact, byte-identical, "held to the same parity and fingerprint gates as the atlas generator" — that is code, not judgment.
- **Refuted alternative:** Express the advisory as prompt instructions like Story 1's filing gate — rejected because a model can't guarantee byte-identical, integer-exact output across runs.

## 2026-07-19 — Reuse the atlas's `buildAdjacency` instead of duplicating it
- **Choice:** Export `buildAdjacency` from `generate-atlas.ts` and import it into `drift-advisory.ts`.
- **Why:** Invariant 9 wants the advisory and atlas to agree on the store's graph; sharing the constructor guarantees it.
- **Refuted alternative:** A standalone copy inside `drift-advisory.ts` — rejected as a divergence risk the invariant explicitly guards against.

## 2026-07-19 — Staleness alarm suppresses low-priority notes too
- **Choice:** When the alarm fires, `renderAdvisory` drops misfiles, refinements, AND low-priority notes, keeping only new-domain candidates.
- **Why:** AC5 says "in place of page-by-page flags"; a note is a page-by-page flag. Invariant 8 names only misfile+refinement, so this resolves the gap.
- **Refuted alternative:** Keep the notes visible under the alarm — rejected as contradicting "in place of page-by-page flags".

## 2026-07-19 — Integer-exact CNM merge gain for community detection
- **Choice:** Compare merges by `twoM*eAB - sigmaTot(A)*sigmaTot(B)`, tie-break by the pair's lexicographically smallest (minSlug, maxSlug) representative tuple.
- **Why:** Byte-identity under the parity gate forbids floating point (decision-record ADDRESS risk).
- **Refuted alternative:** An off-the-shelf Louvain/Leiden — rejected: their randomized visitation can't be held to byte-identity cheaply.

## 2026-07-19 — Harden both CLI self-invoke guards against cross-bundle contamination (deviation from the plan's "no other change" to generate-atlas.ts)
- **Choice:** `drift-advisory.ts` importing `buildAdjacency`/`loadConceptPages`/`registryPath` from `generate-atlas.ts` causes esbuild to inline the whole of `generate-atlas.ts` — including its own top-level `if (import.meta.url === file://${process.argv[1]}) main();` guard — into the `drift-advisory` bundle. After bundling, both files' guards share one `import.meta.url`, so running the vendored `drift-advisory.mjs` was actually executing `generate-atlas`'s `main()` first (it `process.exit`s before `drift-advisory`'s own guard ever runs) — confirmed by the drift-advisory parity test failing with atlas's "Atlas written: ..." stdout instead of the advisory's markdown. Fixed by adding a filename check (`path.basename(process.argv[1]).startsWith("generate-atlas")`, resp. `"drift-advisory"`) alongside the existing `import.meta.url` check in both files' guards — stays true only when that file is the one actually invoked, in both `tsx` (source) and vendored-bundle (`.mjs`) form. Applied the same hardening to `drift-advisory.ts`'s own guard pre-emptively, since Story 3's planned seed driver will import `detectCommunities` and the shared graph helpers from `drift-advisory.ts` and would hit the identical bug otherwise.
- **Why:** The plan's Step 2 ground truth states `generate-atlas.ts`'s only change is the `export` keyword and that "behavior identical → the atlas parity byte-identity tests still pass; only the bundle bytes change" — this held for the atlas's own *output*, but did not anticipate that importing a module with a live self-invoking CLI guard causes esbuild to inline that guard's full call graph into every importer's bundle, corrupting the importer's own CLI behavior. Without this fix, Decision 2.2 (reuse `buildAdjacency` verbatim, do not duplicate) cannot actually work: the drift-advisory bundle would silently run the atlas generator instead of the advisory. This is not a decision-record conflict (the record is silent on bundler internals) — it is a necessary correctness fix to make the plan's own binding decision (2.2) function.
- **Refuted alternative 1:** Split `generate-atlas.ts` into a side-effect-free library file plus a new thin `generate-atlas-cli.ts` entry point (letting esbuild tree-shake the unused CLI code out of `drift-advisory`'s bundle). Rejected as a larger structural change than necessary — it would also require repointing `build-bundles.ts`'s `"generate-atlas"` entry to a new file, a bigger diff than the plan anticipated for a fix that a two-token guard condition already resolves.
- **Refuted alternative 2:** Revert Decision 2.2 and duplicate the ~12-line `buildAdjacency` function locally inside `drift-advisory.ts` instead of importing it. Rejected because the plan explicitly marks 2.2 as an already-made judgment call ("do not re-decide") in service of Invariant 9's literal-agreement requirement, and the guard-hardening fix preserves that reuse while being a strictly smaller, behavior-preserving change (verified: the existing atlas parity tests, which compare rendered *output* bytes rather than source bytes, still pass unchanged).
