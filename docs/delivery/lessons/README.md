# Delivery Lessons

Home for process and delivery lessons captured by `/nxs.close` at the end of an epic. These are
**process/delivery** learnings (how the work went) — distinct from the `LibraryDelta` knowledge
that crosses into the concept library.

## Convention

- **One file per lesson.**
- **Filename:** `<date>-<slug>.md` — e.g. `2026-06-28-stage-gate-timing.md`. Date is the day the
  lesson was recorded (`YYYY-MM-DD`); slug is a short kebab-case summary.
- **Frontmatter:** each lesson carries the source epic.

```markdown
---
date: 2026-06-28
source_epic: <epic-id-or-slug>
tags: [process, delivery]
---

# <Lesson title>

**What happened:** ...

**What we learned:** ...

**What to change next time:** ...
```

## Notes

- Keep lessons short and actionable — a lesson nobody reads costs context for nothing.
- `/nxs.close` writes here automatically; you can also add lessons by hand using the format above.
